# Step05-保存失败反馈与重试状态机

- 日期：`2026-04-07`
- 版本：`0.1.0 / Unreleased`
- 阶段：`Step 05 / Wave-B / 第二十七轮推进`
- 范围：`保存链可靠性升级`

## 1. 变更摘要

本轮将保存失败后的反馈与重试流程正式收敛为显式状态机，并引入统一的 save feedback 模型，解决了“store 能重试，但编辑器、横幅和文案系统无法共享同一份保存语义”的结构问题。

## 2. 已交付内容

1. `noteWorkspaceSaveFeedback.ts` 新增统一保存反馈模型与状态机迁移 helper。
2. `NoteSaveState` 新增 `retrying / recovered`，把失败后的重试中和恢复态正式纳入协议。
3. `useNotesWorkspaceStore.ts` 现在显式执行：
   - `dirty -> saving`
   - `error -> retrying`
   - `saving -> saved`
   - `retrying / error -> recovered`
4. `NoteEditorPane.tsx` 与 `NotesWorkspacePage.tsx` 改为消费统一 `saveFeedback` 模型。
5. `NotesWorkspaceErrorBanner.tsx` 支持 retry CTA，并在页面层与共享 `flushDraft` 对齐。
6. `workspace-save-feedback.contract.test.mjs` 新增保存反馈 contract，并接入主门禁。
7. `zh-CN.ts` 与 `en-US.ts` 已同步补齐新增文案。

## 3. 用户与架构收益

1. 保存失败后不再只是看到孤立的“失败”提示，而是具备“失败、重试中、恢复”的连续可见语义。
2. 编辑器保存按钮、状态徽标和顶部错误横幅开始共享同一份反馈模型，减少界面语义漂移。
3. 后续扩展 save queue、自动退避和恢复遥测时，不需要再回收页面与编辑器层的散落逻辑。

## 5. 风险与回滚评估

### 5.1 已控制风险

1. 新 contract 已进入 `test:workspace:contracts`，一旦显式状态机被回退，会直接在主门禁中暴露。
2. 中文与英文新增文案均已补齐，避免交付环境出现中英混杂。
3. retry CTA 继续复用共享 `flushDraft`，没有引入第二套重试执行入口。

### 5.2 尚存风险

1. 当前仍没有真正的 save queue。
2. 自动退避、重试上限和观测事件尚未形成闭环。
3. 切换笔记与危险操作前的刷盘证据仍需继续补齐。
