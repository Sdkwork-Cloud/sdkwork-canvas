# Step06-本地快照接口-2026-04-13

## 版本定位

- 日期：`2026-04-13`
- 波次：`Wave-B / 第三十三轮推进`
- 影响范围：`Step 06 标准化本地快照接口`
- 关联能力：`CP06-3 / 标准化本地快照接口`

## 变更摘要

本轮把 `@sdkwork/canvas-local` 从“具备本地 schema 和恢复入口的能力包”继续推进为“对下游显式暴露标准化本地快照读边界的能力包”。

当前落地事实：

1. 本地快照空值工厂已显式导出。
2. 本地快照标准化 resolver 已显式导出。
3. 只读快照 reader 已显式导出，并提供默认实现。
4. 搜索与同步后续只需要消费 `NotesLocalWorkspaceSnapshot`，不再需要触碰 `localStorage` envelope 或历史 raw shape。
5. loader failure / unknown version / corrupted payload 会统一降级为空快照。

## 代码影响

1. `sdkwork-canvas-pc-react/packages/sdkwork-canvas-local/src/index.ts`
   - 新增标准化本地快照 exports
2. `sdkwork-canvas-pc-react/scripts/workspace-local-snapshot.contract.test.mjs`
   - 新增标准化本地快照 contract
3. `sdkwork-canvas-pc-react/package.json`
   - 把新 contract 接入 `test:workspace:contracts`
4. `sdkwork-canvas-pc-react/scripts/package-scripts-contract.test.mjs`
   - 同步冻结脚本门禁

## 风险控制

1. 通过标准化快照 contract 固定 reader / resolver / empty snapshot factory 的公开边界。
2. 通过 schema contract 回归验证，确认新增读侧抽象没有破坏 `versioned envelope` 存储约束。
3. 通过 recovery contract 回归验证，确认恢复入口继续消费同一份 `drafts` 语义。
4. 通过根级 `typecheck` 把新 contract 纳入真实主门禁。
