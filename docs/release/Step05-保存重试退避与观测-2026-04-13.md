# Step05-保存重试退避与观测

- 日期：`2026-04-13`
- 版本：`0.1.0 / Unreleased`
- 阶段：`Step 05 / Wave-B / 第二十九轮后续增量`
- 范围：`保存链可靠性升级`

## 1. 变更摘要

本轮在既有 `save queue` 串行保存编排基础上，补齐了保存失败后的自动退避、重试上限与最小观测接口，使保存链从“可以串行执行”继续提升到“可以稳定恢复并可被观测”。

## 2. 已交付内容

1. 新增 `noteWorkspaceSaveRetryPolicy.ts`，冻结默认退避策略 `500ms -> 1500ms -> terminal error`。
2. `useNotesWorkspaceStore.ts` 接入 retry policy，失败后进入 `retrying`，恢复成功记录 `recovered`，耗尽后回落 `error`。
3. `canvas-canvas` 正式依赖 `@sdkwork/canvas-observability` 类型边界。
4. 新增 `workspace-save-retry-policy.contract.test.mjs`，并通过 package/root typecheck 进入主门禁。

## 3. 用户与架构收益

1. 网络短暂波动时，保存会自动退避重试，而不是立即终态失败。
2. 自动重试不再无限持续，最终失败可回到显式 `error`，保留人工重试入口。
3. 保存链已有最小 telemetry 接口，后续观测平台可以直接接入，不必回收当前 store/save 主链。
