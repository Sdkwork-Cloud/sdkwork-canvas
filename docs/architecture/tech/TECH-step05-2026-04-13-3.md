# Step05-退出恢复检查点

- 日期：`2026-04-13`
- 版本：`0.1.0 / Unreleased`
- 阶段：`Step 05 / Wave-B / 第三十轮推进`
- 范围：`编辑器与自动保存可靠性升级`

## 1. 变更摘要

本轮把“退出前保底恢复检查点”正式并入当前保存主链。应用现在会在草稿变更时记录本地恢复检查点，在页面隐藏或关闭前先更新检查点再执行共享 `flushDraft`，并在远端确认保存成功后清理对应检查点。

## 2. 已交付内容

1. `@sdkwork/canvas-local` 获得可执行的本地恢复检查点存储实现。
2. `noteWorkspaceExitRecovery.ts` 冻结退出恢复快照结构与状态准入规则。
3. `useNotesWorkspaceStore.ts` 接入 `draft-change capture` 与 `save-success clear`。
4. `noteWorkspaceAutosaveRuntime.ts` 冻结 `capture -> flush` 顺序。
5. `workspace-exit-recovery.contract.test.mjs` 进入根级合同门禁。

## 3. 用户与运维收益

1. 页面关闭前不再只依赖单次远端刷盘，最近草稿已有本地恢复保底。
2. 异常退出 / 崩溃场景也不再完全丢失最近编辑证据。
3. 已确认保存成功的草稿不会错误残留为本地恢复候选。
4. 后续 `Step 06` 可以直接在当前检查点边界上构建恢复体验。

## 6. 风险与遗留项

1. 当前本地检查点仍是“紧急恢复保底层”，不是完整离线权威副本。
2. 恢复提示 UI、恢复确认流程、放弃恢复流程仍待 `Step 06` 落地。
3. 浏览器存储实现后续可升级到更强的结构化本地存储。
