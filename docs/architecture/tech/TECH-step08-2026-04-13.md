# Step08-主写入路径接入-收口-2026-04-13

本轮继续停留在 `Step 08 / CP08-3`，但目标不再是追加单一写路径，而是把 note 级主写入链真正收口。

本轮新增事实：

- `@sdkwork/canvas-sync` 已新增独立 `permanent-delete` operation。
- `deleteNotePermanently()` 已接入 `note/permanent-delete/queued`。
- `toggleFavorite()` 的 clean-save 成功路径已接入 `note/upsert/queued`。
- `clearTrash()` 已改为在一次 queue snapshot 中批量追加多条 `note/permanent-delete/queued` 任务。

这使 `CP08-3` 当前已经覆盖：

- `createNote`
- `persistActiveNote`
- `toggleFavorite`
- `moveNoteToTrash`
- `restoreNoteFromTrash`
- `moveNote`
- `deleteNotePermanently`
- `clearTrash`

本轮不包含：

- queue worker
- 远端回执应用
- 冲突恢复 UI
- 离在线切换验证

当前结论：

- `CP08-3 / 主写入路径接入 = L4`
- `Step 08` 整体仍为 `L2`

下一轮执行入口：切换到 `Step 08 / CP08-4`
