# Step 04 读策略注册表收敛评审

- 日期：2026-04-07
- Step：04
- 评审主题：repository 未来读策略注册表与按 key 选策略能力收敛
- 当前成熟度：`L3`

## 1. 评审目标

为 `noteRepository.ts` 建立真实可用的“策略注册 -> key 解析 -> 默认回退”边界，避免工作区读取路径继续被硬编码为单一路径，同时为后续 `read-through-cache / replica-snapshot / queued-sync-snapshot` 演进预留稳定入口。

## 2. 本轮变更

### 2.1 新增读策略注册表模块

- 新增 `sdkwork-canvas-pc-react/packages/sdkwork-canvas-canvas/src/repository/noteWorkspaceReadStrategyRegistry.ts`
- 建立 `createNoteWorkspaceReadStrategyRegistry()` 工厂
- 建立 `defaultKey / listKeys() / resolve()` 统一边界
- 注册表在创建阶段主动拒绝重复 key，并在默认 key 不存在时显式失败

### 2.2 repository 接入按 key 选策略能力

- `AppSdkNoteRepositoryOptions` 新增：
  - `workspaceReadStrategies`
  - `workspaceReadStrategyKey`
- `AppSdkNoteRepository` 构造阶段不再只接受单一默认策略，而是先创建注册表，再按 `workspaceReadStrategyKey` 解析实际使用的策略
- 现有 `workspaceReadStrategy` 注入方式保持兼容，默认行为仍然是 `workspace-snapshot`

### 2.3 工作区读策略键域显式化

- `sdkwork-canvas-pc-react/packages/sdkwork-canvas-canvas/src/types/canvasWorkspace.ts` 新增 `NoteWorkspaceReadStrategyKey`
- 当前显式键域包括：
  - `workspace-snapshot`
  - `read-through-cache`
  - `replica-snapshot`
  - `queued-sync-snapshot`
- 这一步把“未来可能会有更多策略”从口头约定提升为代码层面的稳定类型边界

## 3. 契约与验证

- `sdkwork-canvas-pc-react/scripts/workspace-read-strategy-registry.contract.test.mjs`

## 4. 质量评估

### 4.1 可扩展性

- 评估结论：显著提升
- 说明：repository 已从“只能注入单一策略实例”演进为“支持未来策略注册与按 key 切换”，后续接入本地副本、读穿缓存或排队同步快照时，不需要再改写 repository 主体结构

- 评估结论：通过
