# Step05-统一刷盘入口收敛

- 日期：`2026-04-07`
- 版本：`0.1.0 / Unreleased`
- 阶段：`Step 05 / Wave-B / 第二十六轮推进`
- 范围：`保存链可靠性升级`

## 1. 变更摘要

本轮将手动保存、快捷键保存、页面隐藏刷盘统一收敛到共享 `flushDraft` 入口，并在 autosave 计划层正式引入 `shouldFlush` 语义，解决了“保存入口多处存在但判断条件并不一致”的结构问题。

## 2. 已交付内容

1. `noteWorkspaceAutosave.ts` 新增 `shouldFlush` 字段，明确区分调度 autosave 与立即 flush 的资格。
2. `saveState === 'error'` 现在被定义为：允许立即 flush，但不再触发延迟 autosave 重新调度。
3. `NotesWorkspacePage.tsx` 中的 `NoteEditorPane onSave`、页面命令运行时 `persistActiveNote`、`pagehide/visibilitychange(hidden)` 全部共享 `flushDraft`。
4. 新增 `workspace-save-flush-boundary.contract.test.mjs`，并接入主门禁。

## 3. 用户与架构收益

1. 保存失败后的再次保存不再被错误绑定到“页面隐藏”条件。
2. 编辑器手动保存、快捷键保存和隐藏态刷盘开始共享同一条执行入口，保存链语义更加稳定。
3. 后续扩展 save queue、失败重试和危险操作前刷盘时，不需要再回收多个不同入口。

## 5. 风险与回滚评估

### 5.1 已控制风险

1. 新 contract 已进入 `test:workspace:contracts`，保存入口一旦分裂会直接在主门禁中暴露。
2. `error` 状态的 flush 语义已显式冻结，避免失败后无法重试。

### 5.2 尚存风险

1. 当前仍没有真正的 save queue。
2. 失败重试与恢复提示仍未形成可视状态机。
3. 切换笔记与危险操作前的刷盘证据仍需继续补齐。
