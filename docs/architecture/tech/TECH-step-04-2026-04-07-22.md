# Step 04 页面动作编排收敛 - 2026-04-07

## 1. 本轮目标

继续收敛 `NotesWorkspacePage.tsx` 的页面动作边界，将以下“动作解析逻辑”从页面组件中剥离出来：

- 快捷键到页面命令的映射
- 命令面板 action 到页面命令的映射
- 确认框 pending dialog 到页面命令的映射

本轮目标不是把所有页面执行逻辑都抽空，而是先把“条件判断与动作归一化”从页面中拆出去，形成统一 command 解析层。

## 2. 实施范围

本轮实际落地文件如下：

- `sdkwork-canvas-pc-react/packages/sdkwork-canvas-canvas/src/services/noteWorkspacePageActions.ts`
- `sdkwork-canvas-pc-react/packages/sdkwork-canvas-canvas/src/services/index.ts`
- `sdkwork-canvas-pc-react/packages/sdkwork-canvas-canvas/src/pages/NotesWorkspacePage.tsx`
- `sdkwork-canvas-pc-react/scripts/workspace-page-actions.contract.test.mjs`
- `sdkwork-canvas-pc-react/package.json`
- `sdkwork-canvas-pc-react/scripts/package-scripts-contract.test.mjs`

## 3. 设计结论

### 3.1 新增页面动作解析服务

新增 `noteWorkspacePageActions.ts`，输出以下能力：

- `resolveNotesWorkspaceHotkeyCommand()`
- `resolveNotesWorkspaceCommandPaletteCommand()`
- `resolveNotesWorkspaceDialogConfirmCommand()`

统一命令模型为 `NotesWorkspacePageCommand`，页面层只消费标准化命令，不再直接维护多套 if/else 分支。

### 3.2 已收敛的动作类别

#### 快捷键

- `Cmd/Ctrl + K`
- `Cmd/Ctrl + N`
- `Cmd/Ctrl + Enter`
- `Cmd/Ctrl + Shift + F`
- `Cmd/Ctrl + Shift + S`
- `Cmd/Ctrl + Shift + I`
- `Escape`

#### 命令面板

- `create-note`
- `toggle-sidebar`
- `toggle-inspector`
- `focus-search`
- `clear-search`
- `navigate-account`
- `change-view`
- `open-folder`
- `open-note`

#### 确认框

- `clearTrash`
- `deleteNote`
- `deleteFolder`

## 4. 关键行为约束

本轮特别锁定了一个真实差异点：

- 快捷键 `Escape` 清空搜索后需要失焦
- 命令面板 `clear-search` 清空搜索后需要重新聚焦搜索框

该差异已通过合同测试显式建模，避免页面收敛过程中引入行为回归。

## 6. 评估结论

本轮已将页面层的“动作解析”从 UI 组件中抽离为纯服务，明显降低 `NotesWorkspacePage.tsx` 的判断复杂度，但 **Step 04 仍维持 `L3`**，原因如下：

- 页面命令的实际执行仍在 page 层
- repository 读策略抽象仍未建立
- 页面层尚未形成更完整的 command executor / workflow coordinator

## 7. 下一轮建议

1. 将 page command 的执行继续下沉到可测试执行器，进一步减薄页面组件。
2. 开始补 repository 读策略接口，为 Step 06/08 打通扩展面。
