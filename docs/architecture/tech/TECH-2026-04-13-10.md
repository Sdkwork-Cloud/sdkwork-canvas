# 最新版本说明 - 第四十二轮推进补记 - 2026-04-13

本轮继续推进 `Step 08`，但仍然没有把范围扩到 worker 或冲突 UI，而是继续沿 `CP08-3 / 主写入路径接入` 做最小闭环增量：把 `persistActiveNote` 接入同步队列。

当前已经兑现的事实是：

- 活动笔记正文保存成功后，`canvas-canvas` 会向 `@sdkwork/canvas-sync` 写入一条 `note / upsert / queued` 同步任务。
- `createNote` 与 `persistActiveNote` 现在共用 `enqueueNoteUpsertSyncTask(noteId, updatedAt)`，`note/upsert` 入队映射开始稳定收敛。
- `persistActiveNote` 的任务时间戳取自本地提交草稿 `requestedActiveNote.updatedAt`，而不是远端保存响应时间。
- `workspace-sync-write-path.contract.test.mjs` 已同时覆盖 `createNote` 与 `persistActiveNote` 两条主写入队列合同。

本轮仍不宣称 `CP08-3` 或 `Step 08` 完成。因为删除、恢复、永久删除、移动等其余主写入路径尚未接入，后台同步 worker、冲突恢复和离在线切换验证也还没有交付。下一轮执行入口继续保持在 `Step 08 / CP08-3`，优先接入 `moveNoteToTrash`。
