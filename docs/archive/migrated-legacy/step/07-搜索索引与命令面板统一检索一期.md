# Step 07 - 搜索索引与命令面板统一检索一期

## 1. 目标与范围

本 step 用于建立 `canvas-search` 能力层，让顶部搜索、命令面板、最近访问和快速跳转开始共享统一检索语义和统一索引源。

### 1.1 执行输入

- Step 06 的本地快照能力
- `docs/架构/07`
- 当前搜索、命令面板和最近视图实现

### 1.2 本步非目标

- 不在本 step 内实现语义搜索和 AI 问答检索
- 不在本 step 内实现最终版本的多端统一搜索

### 1.3 最小输出

- 本地索引构建器
- 搜索 API 与命令面板统一候选源
- 检索性能基线

## 2. 架构对齐

- `docs/架构/04-技术选型与可扩展策略.md`
- `docs/架构/07-性能-离线-搜索-同步设计.md`

## 3. 当前现状与问题

当前搜索和命令面板建立在“已加载快照”之上，无法支撑：

- 10k+ 规模低延迟检索
- 标题、正文、标签、文件夹统一检索
- 本地离线检索

## 4. 设计

### 4.1 索引模型

- 标题、正文、摘要、标签、文件夹、更新时间、类型、收藏状态统一标准化
- 命令面板和搜索入口共享同一查询服务

### 4.2 UI 收口原则

- 顶部搜索、命令面板、快速跳转共享搜索服务
- 页面只做查询编排和展示，不直接实现搜索算法

## 5. 实施落地规划

1. 增加 `canvas-search` 或等价模块
2. 构建标准化索引文档模型
3. 提供统一查询 API、排序和高亮策略
4. 接入顶部搜索与命令面板

## 6. 测试计划

- 索引构建单测
- 搜索结果排序和过滤单测
- 命令面板与搜索入口集成测试
- 10k 规模性能基准测试

## 7. 结果验证

完成后必须满足：

- 搜索与命令面板开始共享索引语义
- 检索结果不再只依赖页面已加载快照
- 可输出可重复的性能基线

- `CP07-1`：索引文档模型冻结

- `CP07-3`：顶部搜索与命令面板完成接入
- `CP07-4`：性能基线测试通过

### 8.6 本 Step 完成后必须兑现的架构能力

- 统一索引 schema、统一查询服务、统一候选源全部冻结
- 顶部搜索、命令面板、快速跳转开始共享同一套检索语义
- 10k 规模下索引构建、查询和重建都具备可重复验证基线

### 8.7 最快完成的并行执行顺序

1. 先冻结索引文档模型和查询返回模型
2. 再并行推进索引构建器与查询 API
3. 然后并行接入顶部搜索和命令面板
4. 最后统一跑性能基线、准确性回归和文档回写

## 9. 风险与回滚

### 9.1 风险

- 若索引 schema 不稳定，后续同步与版本历史都会返工

### 9.2 回滚

- 保留旧本地过滤入口一段时间，逐步切换到统一查询服务

## 10. 完成定义

- 统一索引与查询服务已落地
- 搜索入口和命令面板开始共享查询语义
