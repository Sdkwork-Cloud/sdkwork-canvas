# Step 04 页面命令执行收敛评审

- 日期：2026-04-07
- Step：04
- 评审对象：`sdkwork-canvas-pc-react/packages/sdkwork-canvas-canvas/src/pages/NotesWorkspacePage.tsx`
- 增量主题：页面命令执行收敛
- 当前成熟度：`L3`

## 1. 本轮目标

将 `NotesWorkspacePage.tsx` 中的页面级命令执行分发逻辑从视图层抽离到可测试的服务边界，避免快捷键、命令面板、确认对话框三条入口各自直接持有分支执行逻辑。

## 2. 落地结果

### 2.1 新增服务边界

- 新增 `packages/sdkwork-canvas-canvas/src/services/noteWorkspacePageCommandExecutor.ts`
- 统一承接以下页面命令：
  - 搜索焦点切换
  - 命令面板打开
  - 新建笔记
  - 保存当前笔记
  - 侧边栏与检查器切换
  - 账户页导航
  - 视图切换
  - 文件夹打开
  - 笔记打开
  - 清空回收站
  - 永久删除笔记
  - 删除文件夹

### 2.2 页面职责收敛

- `NotesWorkspacePage.tsx` 不再直接承载命令 `switch/if` 分发
- 页面层仅保留：
  - React 状态与 DOM 引用
  - store/app store 依赖绑定
  - 服务回调注入
  - 具体 UI 组件渲染

### 2.3 总门禁补齐

- 新增契约测试 `sdkwork-canvas-pc-react/scripts/workspace-page-command-executor.contract.test.mjs`
- 将该契约纳入 `sdkwork-canvas-pc-react/package.json` 的 `test:workspace:contracts`
- 同步更新 `sdkwork-canvas-pc-react/scripts/package-scripts-contract.test.mjs`

## 3. 验证结果

本轮已完成并通过以下验证：

- `node --test --experimental-test-isolation=none scripts/workspace-page-command-executor.contract.test.mjs`
- `node --test --experimental-test-isolation=none scripts/package-scripts-contract.test.mjs`
- `pnpm.cmd --filter @sdkwork/canvas-canvas typecheck`
- `pnpm.cmd test:workspace:contracts`
- `pnpm.cmd typecheck`

验证过程中发现并修复一项真实问题：

- `NotesWorkspacePage.tsx` 中 `handleWorkspacePageCommand` 的 `command` 参数缺少显式类型，导致 `@sdkwork/canvas-canvas` 包级类型检查失败

## 4. 质量评估

### 4.1 架构边界评估

- 评估结论：通过
- 说明：页面命令的“解析”和“执行”边界已拆分为两个独立服务，页面层从执行分发中退出，符合高内聚、低耦合原则

### 4.2 可测试性评估

- 评估结论：通过
- 说明：命令执行逻辑已可以通过依赖注入回调进行纯契约验证，不再依赖 React 页面渲染环境

### 4.3 维护性评估

- 评估结论：通过
- 说明：后续新增页面命令时，优先在服务层扩展命令类型和执行器，不需要继续膨胀页面组件分支

### 4.4 完整性评估

- 评估结论：部分通过
- 说明：页面命令执行已抽离，但页面层仍承担较多依赖装配职责；repository 侧也尚未完成可扩展读取策略边界

## 5. 剩余差距

Step 04 仍维持 `L3`，当前剩余主阻塞点如下：

1. `NotesWorkspacePage.tsx` 仍持有较重的命令依赖装配逻辑，尚未完全收敛为更薄的页面适配层
2. `noteRepository.ts` 仍缺少面向 `read-through / replica / sync` 的读取策略抽象，未来离线与同步能力扩展点尚未建立

## 6. 结论

本轮增量已经真实落地并完成验证，页面命令执行职责完成一次关键收敛；但从 Step 04 的总体目标判断，当前仍属于“页面与工作区编排正在收口、尚未完全闭环”的 `L3` 阶段，不能提前进入 Step 05。
