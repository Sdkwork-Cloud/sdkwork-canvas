# Step 04 创建文件夹写路径编排收敛

- 日期：`2026-04-07`
- Step：`04-工作区边界收敛与数据访问抽象`
- 波次：`Wave-B / 第十三轮推进`
- 当前等级：`L3`

## 1. 本轮目标

在上一轮完成 `createNote` 写路径下沉之后，本轮继续沿同一模式处理 `createFolder()`。目标仍然不是扩展功能，而是继续压缩 `useNotesWorkspaceStore.ts` 内的写路径编排残留。

本轮选 `createFolder()` 的原因：

1. 它仍完全留在 Store 中。
2. 协作链最短，适合做连续的小步收敛。
3. 完成后能把 Step 04 的 create 系列写路径统一到同一个 coordinator 内。

## 2. TDD 执行记录

### 2.1 先写失败契约

先扩展 `sdkwork-canvas-pc-react/scripts/workspace-write-path.contract.test.mjs`，新增以下契约：

- `createNotesWorkspaceWriteCoordinator()` 必须暴露 `createFolderState()`。
- `createFolderState()` 必须调用远程创建。
- 返回结果必须包含：
  - `createdFolderId`
  - `folders`
  - `expandedFolderIds`

红灯验证命令：

```powershell
node --test --experimental-test-isolation=none scripts/workspace-write-path.contract.test.mjs
```

首次失败原因：

- `coordinator.createFolderState is not a function`

失败原因正确，说明本轮锁定的是“createFolder 编排接口缺失”，不是测试噪音。

### 2.2 绿色实现

随后在 `packages/sdkwork-canvas-canvas/src/services/noteWorkspaceWriteCoordinator.ts` 中补齐：

1. `createFolder` 依赖注入。
2. `NoteWorkspaceCreateFolderStateResult`。
3. `createFolderState()`。

实现逻辑：

1. 调用 `createFolder(name, parentId)`。
2. 失败时返回统一错误结果。
3. 成功时通过 `planCreatedFolderState()` 返回排序后的文件夹列表与展开状态。

### 2.3 Store 接线

在 `packages/sdkwork-canvas-canvas/src/store/useNotesWorkspaceStore.ts` 中：

- `workspaceWriteCoordinator` 新增 `createFolder` 依赖。
- `createFolder()` 改为消费 `createFolderState()`。
- Store 不再直接内联 `workspaceService.createFolder()` 与 `planCreatedFolderState()`。
