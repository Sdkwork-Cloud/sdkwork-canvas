# Step 04 创建笔记写路径编排收敛

- 日期：`2026-04-07`
- Step：`04-工作区边界收敛与数据访问抽象`
- 波次：`Wave-B / 第十二轮推进`
- 当前等级：`L3`

## 1. 本轮目标

本轮不扩张能力面，只继续压缩 `useNotesWorkspaceStore.ts` 中仍然残留的写路径编排逻辑。优先选择 `createNote()`，因为它同时承担了以下四类职责：

1. 调用 `workspaceService.save()` 发起创建。
2. 调用 `workspaceService.findById()` 回填详情。
3. 调用 `planCreatedNoteState()` 回放状态变更。
4. 在 store 内部拼装错误分支和返回值。

这段逻辑仍然属于 Step 04 需要继续下沉的“写路径编排残留”，但粒度足够小，适合作为本轮最小真实增量。

## 2. TDD 执行记录

### 2.1 先写失败契约

先扩展 `sdkwork-canvas-pc-react/scripts/workspace-write-path.contract.test.mjs`，增加以下契约：

- `createNotesWorkspaceWriteCoordinator()` 必须对外暴露 `createNoteState()`。
- `createNoteState()` 必须先持久化、再回填详情、再返回创建后的状态计划。
- 返回结果必须包含 `createdNoteId / activeNoteId / activeNote / selectedFolderId / expandedFolderIds / canvas / trashedNotes`。

红灯验证命令：

```powershell
node --test --experimental-test-isolation=none scripts/workspace-write-path.contract.test.mjs
```

首次失败原因：

- `workspaceWritePathModule.createNotesWorkspaceWriteCoordinator is not a function`

失败原因与预期一致，说明测试锁定的是“缺少写路径编排边界实现”，而不是测试自身错误。

### 2.2 绿色实现

随后在 `packages/sdkwork-canvas-canvas/src/services/noteWorkspaceWriteCoordinator.ts` 中实现：

- `createNotesWorkspaceWriteCoordinator()`
- `createNoteState()`

实现职责边界如下：

1. 接收 `saveNote / findNoteDetail` 依赖。
2. 先执行创建持久化。
3. 失败时返回统一错误结果。
4. 成功时回填详情；如果详情缺失，则回落到创建摘要 + 输入内容。
5. 最后统一通过 `planCreatedNoteState()` 生成状态计划。

### 2.3 Store 接线

在 `packages/sdkwork-canvas-canvas/src/store/useNotesWorkspaceStore.ts` 中：

- 移除 `createNote()` 内部对 `workspaceService.save()` 与 `workspaceService.findById()` 的直接编排。
- 改为消费 `createNotesWorkspaceWriteCoordinator().createNoteState()`。
- store 仅保留：
  - 脏草稿前置持久化守卫
  - 结果写回
  - `saveState / errorMessage` 这类状态层职责
