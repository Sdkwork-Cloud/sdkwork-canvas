# Review 文档索引

本目录用于沉淀 `sdkwork-canvas` 在执行 `docs/step/` 过程中的审计、执行卡、差距矩阵、能力兑现、复盘和波次总验收证据。

## 1. 目录目标

- 记录每个 Step 的真实输入、输出、风险、验证和结论
- 作为 `docs/step/` 与 `docs/架构/` 之间的执行证据层
- 为 `docs/release/` 提供版本侧可追溯材料
- 避免“代码已经变化，但无人能证明当前到底完成到哪里”的信息断层

## 2. 命名建议

- `step-XX-执行卡-YYYY-MM-DD.md`
- `step-XX-质量审计-YYYY-MM-DD.md`
- `step-XX-能力兑现-YYYY-MM-DD.md`
- `step-XX-差距矩阵-YYYY-MM-DD.md`
- `step-XX-复盘-YYYY-MM-DD.md`
- `wave-X-总验收-YYYY-MM-DD.md`

## 3. 使用规则

- 每个 Step 至少要有执行卡、质量审计和能力兑现三类文档中的适用项
- 涉及现状盘点、结构梳理或差距冻结时，必须补充差距矩阵
- 波次完成后必须补充总验收
- 所有日期统一使用绝对日期格式：`YYYY-MM-DD`
- 所有结论都必须引用真实代码、脚本、workflow 或验证命令，不得只写主观判断

## 4. 当前已建立基线

截至 `2026-04-07`，本目录已经开始承接：

- `Step 00` 的执行门禁冻结证据
- `Step 01` 的现状基线审计与差距矩阵证据

后续所有 Step 应继续沿用同一命名和审计口径。
