# 最新版本说明 - 第四十三轮推进补记 - 2026-04-13

本轮继续推进 `Step 08`，仍然没有把范围扩到 worker 或冲突 UI，而是继续沿 `CP08-3 / 主写入路径接入` 做最小闭环增量：把 `moveNoteToTrash` 接入同步队列。

当前已经兑现的事实是：

- 笔记移入废纸篓成功后，`canvas-canvas` 会向 `@sdkwork/canvas-sync` 写入一条 `note / delete / queued` 同步任务。
- 队列写入逻辑已收敛为 `enqueueNoteSyncTask(noteId, operation, atValue)`，`createNote`、`persistActiveNote` 与 `moveNoteToTrash` 已开始复用同一条 note 级入队边界。
- `moveNoteToTrash` 的任务时间戳优先取自删除事实 `deletedAt`，而不是普通 `updatedAt`。
- `workspace-sync-write-path.contract.test.mjs` 已同时覆盖 `createNote`、`persistActiveNote` 和 `moveNoteToTrash` 三条主写入队列合同。

本轮仍不宣称 `CP08-3` 或 `Step 08` 完成。因为恢复、永久删除、移动等其余主写入路径尚未接入，后台同步 worker、冲突恢复和离在线切换验证也还没有交付。下一轮执行入口继续保持在 `Step 08 / CP08-3`，优先接入 `restoreNoteFromTrash`。
