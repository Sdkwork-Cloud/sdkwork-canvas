# Step 08 - 队列 worker 运行时调度闭环 - 2026-04-14

## 本轮目标

- 在 `@sdkwork/canvas-sync` 内把 package-local executor 推进为可驱动的 worker runtime。
- 形成“串行 drain -> 重入合并 -> 最早 retry 调度 -> 自动回放 -> 可释放”的最小调度闭环。
- 保持范围收敛，不提前扩散到 UI、desktop runtime 或真实远端 transport。

## 本轮非目标

- 不在本轮接入真实远端 SDK / transport。
- 不在本轮把 runtime 接到 `canvas-canvas` store 或 desktop background 线程。
- 不在本轮交付冲突提示 UI、手动 replay 入口或离在线切换 smoke。

## 实际落地

1. `sdkwork-canvas-pc-react/scripts/workspace-sync-worker-runtime.contract.test.mjs`
   - 新增 4 条 Node contract，冻结以下语义：
   - `requestDrain()` 会串行消费队列，直到没有 runnable task 为止。
   - 重叠的 drain 请求会合并到同一个 active run，不会并发执行多个 worker。
   - runtime 只为最早到期的 `retrying` 任务挂起调度，并在到期后自动回放。
   - `dispose()` 会取消未到期的 retry timer。
2. `sdkwork-canvas-pc-react/packages/sdkwork-canvas-sync/src/index.ts`
   - 新增 `createNotesSyncWorkerRuntime(...)`。
   - 新增 `NotesSyncWorkerScheduler`、`CreateNotesSyncWorkerRuntimeOptions`、`NotesSyncWorkerRuntime`。
   - 默认 scheduler 复用 `setTimeout / clearTimeout`，允许测试和后续 runtime 注入自定义调度器。
   - runtime 复用既有 `executeNextNotesSyncTask(...)`，没有复制第二套状态机逻辑。
3. `sdkwork-canvas-pc-react/package.json`
   - 根级 `test:workspace:contracts` 已接入 `workspace-sync-worker-runtime.contract.test.mjs`。
4. `sdkwork-canvas-pc-react/scripts/package-scripts-contract.test.mjs`
   - 已同步冻结新的根级合同脚本编排。

- executor / runtime 仍未接入 `canvas-canvas` 或 desktop runtime 的真实调度边界。
- 仍缺真实远端 handler 与回执应用链路。
- 仍缺 conflict/manual replay UI 与用户恢复入口。
- 仍缺离在线切换、断网恢复与冲突演练 smoke。
