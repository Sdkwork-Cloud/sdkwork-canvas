# Step04-创建笔记写路径编排收敛

- 发布日期：`2026-04-07`
- 发布阶段：`Unreleased`
- 关联 Step：`Step 04`
- 当前等级：`L3`

## 1. 发布摘要

本轮将 `createNote` 的工作区写路径从 Store 内嵌编排下沉到独立的 write coordinator。该调整不改变用户可见功能，但显著提升以下工程能力：

1. 写路径边界清晰度。
2. store 职责纯度。
3. contract 可测试性。
4. 后续 `createFolder / renameFolder / moveNote` 继续收敛的可复制性。

## 2. 代码变更

### 新增/增强

- `packages/sdkwork-canvas-canvas/src/services/noteWorkspaceWriteCoordinator.ts`
  - 新增 `createNotesWorkspaceWriteCoordinator()`
  - 新增 `createNoteState()`

### 调整

- `packages/sdkwork-canvas-canvas/src/store/useNotesWorkspaceStore.ts`
  - `createNote()` 不再直接内联 `save + findById + planCreatedNoteState`
  - 改为消费 write coordinator 返回的结果

- `sdkwork-canvas-pc-react/scripts/workspace-write-path.contract.test.mjs`
  - 新增 create note 写路径编排 contract

## 4. 风险与结论

### 已降低的风险

1. `createNote` 协作链不再散落在 Store 中，后续改动更可控。
2. 创建结果的详情回填与状态计划可独立验证。

### 尚存风险

1. `createFolder / renameFolder / moveNote` 仍未按同样模式完全下沉。
2. repository 策略层尚未进入更高成熟度设计。

### 当前结论

- `Step 04` 继续推进中。
- 本轮是真实工程增量，但不足以将 Step 04 从 `L3` 提升到 `L4`。
