# 12. Step 03 L4 收口记录 - 2026-04-07

## 12.1 目标

记录 Step 03 从“会话安全与配置治理待收口”推进到 `L4` 的最终事实，避免后续继续沿用“会话仍主要落在 localStorage”或“环境配置优先级未冻结”的过期结论。

## 12.2 收口原因

Step 03 的原始目标是解决三类基础风险：

- 会话令牌长期散落在低安全级浏览器持久层
- 运行时配置来源、覆盖顺序和 owner scope 缺乏权威定义
- Desktop 与 Web 在会话持久化和启动期水合路径上缺少统一边界

本轮已完成以下收口动作：

- `canvas-core`
  - 新增统一会话存储适配接口 `configureAppSdkSessionStoreAdapter`
  - 新增运行时配置解析入口 `resolveAppSdkRuntimeContext`
  - Web 默认会话存储从 `localStorage` 切换为 `sessionStorage`
  - 保留旧 `localStorage` 会话键的一次性迁移能力，并在迁移后立即清理旧键
- `canvas-desktop`
  - 新增 `sessionBridge.ts`，把桌面前端会话镜像与 Tauri 原生会话状态串行同步
  - Desktop 启动期在 `createDesktopApp()` 中安装原生会话桥
  - Tauri 新增 `read_session_state / write_session_state / clear_session_state` 命令
  - 原生权威副本冻结为 app data 目录中的 `desktop-session.json`
- 环境配置
  - `.env.example`
  - `.env.development`
  - `.env.test`
  - `.env.production`
  - 以上文件已显式声明 `VITE_APP_OWNER_MODE=tenant`

- Step 03 的历史阻塞项已全部解除。
