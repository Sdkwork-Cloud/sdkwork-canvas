# Step 08 - 队列 worker 最小执行闭环 - 2026-04-14

## 本轮目标

- 从 `CP08-3 / 主写入路径接入` 正式切换到 `CP08-4 / 冲突与失败恢复验证`。
- 在 `@sdkwork/canvas-sync` 包内交付最小可验证的 worker 执行链，而不是提前扩散到 UI、desktop runtime 或远端 transport。

## 本轮非目标

- 不在本轮接入真实远端写入 SDK。
- 不在本轮交付冲突提示 UI、人工恢复交互或离在线切换 smoke。
- 不在本轮改写 `canvas-canvas` 的主写入链路。

## 实际落地

1. `sdkwork-canvas-pc-react/scripts/workspace-sync-worker.contract.test.mjs`
   - 新增 5 条 Node contract，冻结以下语义：
   - oldest queued task 必须先持久化为 `running` 再执行。
   - 到期 `retrying` 任务必须先释放回 `queued` 再自动重放。
   - handler 结果必须回写为 `completed / conflict / retrying / failed`。
   - 成功回执必须能够刷新 `remoteCursor`。
2. `sdkwork-canvas-pc-react/packages/sdkwork-canvas-sync/src/index.ts`
   - 新增 `executeNextNotesSyncTask(...)` 最小 executor API。
   - executor 会先释放 `nextRetryAt <= execution.at` 的任务，再按 `enqueuedAt` 选择 oldest runnable queued task。
   - executor 会在调用注入 handler 前先回写 `running`，并在 handler 返回后回写最终状态。
3. `sdkwork-canvas-pc-react/package.json`
   - `test:workspace:contracts` 已接入 `workspace-sync-worker.contract.test.mjs`，不再依赖人工单独执行。
4. `sdkwork-canvas-pc-react/scripts/package-scripts-contract.test.mjs`
   - 已同步冻结新的根级合同脚本编排。
