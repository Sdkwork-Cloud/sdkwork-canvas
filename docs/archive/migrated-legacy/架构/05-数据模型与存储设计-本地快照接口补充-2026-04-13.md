# 05-数据模型与存储设计-本地快照接口补充-2026-04-13

- 日期：`2026-04-13`
- 归属 Step：`Step 06 / CP06-3`

## 1. 背景

`CP06-1` 已冻结 `sdkwork-canvas-local-workspace` 的 `version = 1` envelope，`CP06-2` 已冻结恢复入口对 `drafts` 的消费方式。本轮继续补上“对下游显式暴露标准化快照读边界”这一层，避免未来搜索和同步直接耦合本地存储格式。

## 2. 新增模型边界

`@sdkwork/canvas-local` 本轮新增以下显式模型边界：

1. `createEmptyNotesLocalWorkspaceSnapshot()`
2. `resolveNotesLocalWorkspaceSnapshot()`
3. `NotesLocalWorkspaceSnapshotReader`
4. `createNotesLocalWorkspaceSnapshotReader()`
5. `canvasLocalWorkspaceSnapshotReader`

## 3. 建模结论

1. `canvas / folders / drafts` 不再只是本地 schema 字段集合，而是显式、可复用、可 contract 化的快照模型。
2. 下游不应再自己理解 `{ version, workspace }`，也不应直接耦合本地存储 key。
3. legacy raw snapshot、current versioned envelope、loader failure 与 unknown version 都统一收敛到 `NotesLocalWorkspaceSnapshot`。
4. 当前仍不在 `Step 06` 一期实现真实搜索索引或同步状态机，但它们未来的数据输入边界已经被冻结。
