# 最新版本说明 - 第五十三轮推进补记 - 2026-04-14

## 本轮补记

- 已把同步队列任务显式区分为“可回放”和“不可回放”。
- 当前 `direct-write` note 主写入路径产生的任务全部显式标记为 `replayable: false`。
- worker 现在不会误执行这些不安全任务，而是统一终态失败为 `replay-disabled`。

## 已完成

1. `canvas-sync` 为 `NotesSyncTask` 新增 `replayable`，并把缺省与 legacy 读取统一收敛为 `false`。
2. `executeNextNotesSyncTask()` 新增不可回放保护，避免当前同步影子任务重复写远端。
3. `canvas-canvas` 已在 `createNote / persistActiveNote / toggleFavorite / moveNoteToTrash / restoreNoteFromTrash / moveNote / deleteNotePermanently / clearTrash` 路径显式写入 `replayable: false`。
4. `state-machine / queue / write-path / worker / worker-runtime / runtime-boundary / canvas store` 相关测试均已对齐并通过。

## 仍待完成

1. replay-safe transport / idempotency 边界。
2. 真正的 remote ack apply / `remoteCursor` 合并闭环。
3. app/bootstrap 或 desktop/background 的默认 runtime caller。
4. conflict/manual recovery UI。
