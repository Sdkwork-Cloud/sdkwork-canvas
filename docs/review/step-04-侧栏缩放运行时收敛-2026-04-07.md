# Step 04 侧栏缩放运行时收敛评审

- 日期: 2026-04-07
- Step: 04
- 当前能力等级: L3
- 评审范围: sidebar resize pointerdown/pointermove/pointerup 生命周期、宽度钳制、cleanup 幂等性、页面运行时收敛

## 1. 评审结论

本轮增量将 `NotesWorkspacePage.tsx` 中内联的侧栏缩放运行时编排下沉到独立服务 `noteWorkspaceSidebarResizeRuntime.ts`。页面层不再直接负责：

1. 维护 `pointermove` / `pointerup` 生命周期。
2. 对侧栏宽度执行 `220..420` 的范围钳制。
3. 在释放指针或手动终止时执行一次性 cleanup。

页面层现在只保留 `pointerdown` 容器入口与浏览器事件绑定适配，侧栏缩放的运行时职责已经进入可独立验证的 service 边界。

## 2. 设计与实现评审

### 2.1 设计合理性

- 优点: 运行时服务只消费注入依赖，不依赖 React Hook，也不依赖浏览器全局单例，便于 Node contract 测试直接验证。
- 优点: 页面层只保留“事件入口 + 依赖装配”，符合 Step 04 对 page/container 边界收敛的目标。
- 优点: 侧栏宽度钳制规则进入独立 service，后续若引入用户偏好、桌面窗口尺寸协同或布局策略切换，可以在 service 层扩展，而不需要重写页面逻辑。

### 2.2 实现逻辑评审

`noteWorkspaceSidebarResizeRuntime.ts` 统一提供：

1. `startNotesWorkspaceSidebarResize(dependencies)`

行为约束如下：

- 运行时启动后必须统一绑定 `pointermove` 与 `pointerup`。
- 每次移动时都必须基于 `startWidth + deltaX` 计算宽度，并执行 `220..420` 钳制。
- `pointerup` 触发后必须执行 cleanup。
- 返回的 cleanup 必须可手动调用，且多次调用只能生效一次。

### 2.3 架构质量评估

- 高内聚: 宽度计算、边界钳制、事件清理都集中到单一 service，达到。
- 低耦合: 页面只负责注入 `setSidebarWidth` 与 DOM 事件适配函数，达到。
- 易测试: pointer 生命周期、宽度范围和 cleanup 幂等性都能 contract 化，达到。
- 易扩展: 后续接入布局偏好、桌面窗口尺寸联动、埋点统计时具备稳定入口，基本达到。

## 4. 评估标准与达成情况

| 评估项 | 标准 | 结果 |
| --- | --- | --- |
| 运行时边界 | 页面不得直接维护 `pointermove/pointerup` 生命周期 | 达成 |
