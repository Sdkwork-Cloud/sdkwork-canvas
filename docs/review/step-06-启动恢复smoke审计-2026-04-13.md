## 覆盖矩阵

本轮 smoke contract 覆盖以下启动场景：

1. 当前版本 envelope 且活动 live note 存在本地草稿
2. legacy raw snapshot 直接落在 storage 中
3. unknown version payload
4. corrupted payload
5. `localStore.loadWorkspace()` 直接抛错
6. 已进 trash 或远端已不存在的草稿在启动时被过滤

## 审计判断

### 通过项

1. 启动链路已经具备恢复能力，本轮不需要再改生产实现。
2. 现有 `canvas-local` schema 兼容能力能够被启动恢复直接消费。
3. 启动阶段对异常 payload 和读取失败的降级行为稳定，能够回退为空恢复快照。
4. `activeRecoveredDraft` 会跟随启动时解析出的活动 note 正确落位。

### 剩余风险

1. 本轮 smoke 证明的是“启动恢复链路存在且稳定”，不是新的 UI 交互验收。
2. 页面展示、恢复横幅交互和恢复后的编辑器行为，继续由既有 `workspace-local-recovery.contract.test.mjs` 和页面边界 contract 兜底。
