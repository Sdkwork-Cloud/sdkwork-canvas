# Step 04-重命名文件夹写路径编排收敛

## 1. 本轮目标

本轮的目标不是继续扩散功能面，而是继续压缩 `useNotesWorkspaceStore.ts` 中残留的写路径编排职责，把 `renameFolder()` 的远程调用、结果归一化和状态计划拼装下沉到显式 coordinator。

目标判定标准：

1. Store 不再直接组合 `workspaceService.renameFolder()` 与 `planRenamedFolderState()`。
2. 写路径协作链具备独立的 Node contract，可先红灯证明缺口，再转绿证明实现。
3. 变更后不改变当前 Step 的真实分级判断，仍以事实为准维持 `L3`。

## 2. 实施结果

本轮已完成以下落地：

1. 在 `packages/sdkwork-canvas-canvas/src/services/noteWorkspaceWriteCoordinator.ts` 中新增 `NoteWorkspaceRenameFolderStateResult`，并为 coordinator 注入 `renameFolder` 依赖。
2. 新增 `renameFolderState()`，统一承担“远程重命名 -> 返回 id 归一化 -> `planRenamedFolderState()` 拼装 -> 错误结果归一化”的写路径编排职责。
3. 在 `packages/sdkwork-canvas-canvas/src/store/useNotesWorkspaceStore.ts` 中，将 `renameFolder()` 改为只消费 coordinator 返回结果；store 不再直接承接远程调用和 rename plan 组装。
4. 在 `sdkwork-canvas-pc-react/scripts/workspace-write-path.contract.test.mjs` 中新增 `renameFolderState()` contract，并先验证红灯失败原因为 `coordinator.renameFolderState is not a function`，随后完成实现转绿。
